# 项目总体概述

## 1. 项目名称

结合代码中的工程标识 `smartclass-ttbt`、`SmartClassApplication` 以及大量“智慧课堂”命名，可以将本项目表述为：

**SmartClass 智慧课堂 AI 教学系统**

这是一个面向教学与学习场景的后端系统，重点融合了课程学习管理、AI 会话辅助、课程知识库同步、日常英语学习内容和用户交互能力。

## 2. 项目建设目标

从当前代码实现可以看出，项目建设目标并非单点功能开发，而是围绕“智能化学习平台”形成完整业务闭环，具体包括：

- 建立课程管理、章节小节组织、练习与作业提交、学习进度追踪等基础教学能力。
- 提供以 AI 分身为核心的多轮聊天式学习助手，支持角色设定、会话管理、历史记录与流式输出。
- 将 AI 能力嵌入课程学习过程，围绕课程、小节、作业、错题和视频进度构建课程场景学习助手。
- 借助 Dify 工作流与知识库接口，将课程介绍、章节、小节、题目和课程资料同步为可检索知识资源。
- 通过每日文章、每日词汇、词书、收藏、点赞、公告、反馈、好友和私聊等功能增强平台的持续学习属性与用户活跃度。

## 3. 系统解决的问题

### 3.1 传统在线学习系统的问题

传统课程平台通常只能提供静态课程内容、视频播放和简单练习，学生在学习中遇到问题时往往无法即时获得解答，也难以形成基于学习状态的个性化指导。

### 3.2 本系统针对性解决的问题

本项目从代码层面解决了以下几类问题：

- **课程内容组织问题**：通过课程、章节、小节、习题、作业、资料等实体构建结构化教学内容。
- **学习过程记录问题**：通过 `user_progress`、`video_learning_record`、`user_course`、`homework_submission` 等表记录学习状态。
- **AI 辅学脱离场景的问题**：通过 `LearningContextAssembler` 组装课程、章节、小节、进度、错题和作业上下文，使 AI 回答不再停留在通用聊天层面。
- **知识利用率不足的问题**：通过 `course_knowledge_sync` 和 `subject_ai_binding` 将课程内容同步至 Dify 知识库与工作流。
- **用户活跃度不足的问题**：通过每日词汇、每日文章、收藏、点赞、等级经验、成就、公告与社交互动提升持续使用意愿。

## 4. 主要使用角色

从 `UserRoleEnum`、`AuthCheck` 和各控制器权限可以确认系统主要包含以下角色：

- **学生/普通用户**：进行课程学习、练习提交、作业提交、每日词汇学习、每日文章阅读、AI 对话、好友互动等。
- **教师**：管理本人课程、章节、小节、作业批改、课程知识同步、查看课程相关数据。
- **管理员**：管理用户、课程分类、公告、每日内容、AI 分身、教师数据和平台级统计。

在业务上，AI 分身本身不是独立角色，而是由管理员或普通用户创建、配置并供学习场景调用的“智能资源”。

## 5. 核心业务主线

### 5.1 教学业务主线

系统的传统业务主线可以概括为：

**课程创建 -> 章节/小节组织 -> 练习与资料配置 -> 学生学习 -> 视频进度记录 -> 习题/作业提交 -> 进度完成与个人中心展示**

### 5.2 AI 业务主线

系统的 AI 业务主线分为两条：

- **聊天型学习助手主线**：用户选择 AI 分身 -> 创建对话会话 -> 发送消息 -> 调用 Dify 对话接口 -> 保存聊天记录 -> 支持流式输出与会话摘要。
- **课程场景学习助手主线**：用户进入课程学习场景 -> 系统组装课程上下文 -> 根据学科绑定工作流或课程专属知识 -> 调用 Dify 工作流/对话接口 -> 返回更贴合课程内容的回答。

### 5.3 平台增强主线

除课程学习外，系统还建立了：

- 每日文章阅读链路
- 每日词汇学习与复习链路
- 公告通知链路
- 用户反馈链路
- 社交互动与私聊链路
- 经验值、等级、成就等成长链路

这些能力共同构成一个“可学习、可互动、可持续使用”的智慧教学平台，而不是单纯的课程展示系统。
